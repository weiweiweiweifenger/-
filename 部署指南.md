# 文思（WenSi）部署指南

本文档结合《快速启动指南》与项目需求说明，概述系统定位、架构与从环境准备到启动验证的部署步骤。日常开发可优先阅读根目录下的《快速启动指南.md》获取最短路径命令。

---

## 1. 项目概述

### 1.1 背景与定位

在知识工作日益复杂化的今天，文字创作已不再是单纯的线性书写。学术研究者面对的是大量文献与笔记；技术文档撰写者需要维护不断迭代的接口与术语；小说作者需要管理人物、地点与时间线。文思面向这些场景，提供 **Web 应用（浏览器访问）**、**无需安装客户端** 的专业级写作环境。

- **项目名称**：文思（WenSi）  
- **核心定位**：下一代智能写作引擎——面向学术研究者、技术文档撰写者、小说作者、笔记爱好者的专业级写作环境。  
- **Slogan**：写文如写「码」，让灵感一键「运行」。

### 1.2 核心理念与价值

**像写代码一样写作**：将结构化组织、跨文件引用、版本控制、自动化辅助等思路引入文字创作；将编程中的「函数调用」范式转化为通用的「知识引用」机制——在章节中引用知识条目名称或别名，系统支持悬浮预览、跳转原文、反向追溯引用位置。

**产品价值主张（摘要）**：

- 学术：文献与笔记可复用，引用关系可追溯。  
- 技术文档：术语一处维护、全局一致。  
- 小说与创意：设定可引用，降低前后矛盾风险。  
- 笔记：双向链接与知识图谱能力（进阶功能见需求文档）。

### 1.3 技术架构概览

| 层次 | 说明 |
|------|------|
| 后端 | Spring Boot **3.5.13** + JDK 17，分层架构（Controller-Service-Repository），包名 `com.wensi.backend`；Spring Security 用于密码 BCrypt 加密；JPA + MySQL；Flyway 管理增量迁移。 |
| 前端 | 原生 HTML/CSS/JS，无第三方 UI 框架；IDE 风格三栏布局；通过 **HTTP REST** 与后端通信（默认 `http://localhost:8081`）。AI 聊天等为**同步 REST** 返回，非 WebSocket 流式。 |
| 鉴权 | 登录成功后前端将用户信息写入 `localStorage`（`wensi_user`），请求 URL 携带 `userId`；后端 Service 层校验资源归属。当前未实现 JWT 过滤器；`SecurityConfig` 对 HTTP 接口为放行，生产环境需加强鉴权。 |
| 存储 | MySQL 存元数据与关系；**用户正文与目录树** 存于 `wensi.storage.base-path`（默认项目根下 `data/user-data`，UTF-8）。 |
| AI | 用户在「用户中心」配置 Provider（API Key、模型、基地址）；后端经 `AiHttpClient` 调用 OpenAI 兼容接口。RAG 向量索引为按用户/知识库的本地 JSON 文件。 |

**适用场景**：学术写作、学习笔记、技术文档、创意写作、商业写作等。

### 1.4 核心工作流（部署后用户侧）

1. 用户注册/登录，系统在服务端为其创建工作区目录。  
2. 创建「写作项目」或「知识库」，对应服务器目录结构。  
3. 在项目中编辑章节，可引用已链接知识库中的条目。  
4. AI 辅助（摘要、Tab 补全、聊天、记忆管理等）贯穿流程，依赖用户在设置中配置的模型与密钥。

### 1.5 非功能与安全要点（运维需知）

- 用户密码使用 **BCrypt** 存储；AI API Key 保存在数据库，由**后端**调用大模型，前端不直接持 Key。  
- 数据隔离依赖 **`userId` 参数 + Service 层归属校验** + 每用户独立目录；生产环境建议改为 Token/JWT 等标准鉴权，勿长期仅依赖 URL 参数。  
- 生产环境务必通过环境变量注入数据库密码等敏感项，**勿在仓库中明文写死**。  
- 用户文件与数据库需定期备份；删除操作请走应用接口，见下文 1.6 节。

### 1.6 删除、外键级联与磁盘文件（重要）

本系统采用 **「库表元数据 + 磁盘正文」双存储**。MySQL 外键 `ON DELETE CASCADE` **只删表里的行，不会删硬盘上的文件夹**。

#### 方式一：通过应用接口删除（推荐，与线上一致）

| 操作 | 磁盘文件 | 数据库（含级联） |
|------|----------|------------------|
| 删除**项目** | 删除整个 `data/user-data/{userId}/project/{projectId}/`（其下全部章节正文、摘要、备注等） | 删除 `projects` 行后级联：`chapters`、`links_project`、`links_chapter`（随章节）、`short_term_memory`、`long_term_memory`、`chat_sessions` → `chat_messages` 等 |
| 删除**章节** | 删除该章节对应目录（`ChapterService` 中 `deleteDirectory`） | 删除 `chapters` 行；级联 `links_chapter`、相关短期记忆等 |
| 删除**知识库** | 删除整个 `.../knowledge_base/{kbId}/` | 级联 `knowledge`、`links_project`（指向该库的连接）等 |
| 删除**知识条目** | 删除该条目目录 | 级联 `links_chapter`（指向该知识的引用记录）等 |

**删项目时不会发生的事：**

- **不会**删除已连接的**知识库**及其磁盘文件（知识库属于用户，与项目独立；仅删除 `links_project` 中的连接关系）。  
- **不会**自动清理知识库侧的 **RAG 向量索引文件**（索引随知识库维护；删项目不影响知识库索引）。

#### 方式二：仅在数据库中直接 `DELETE`（不推荐）

若在 Navicat/SQL 中直接删除 `projects` 一行：

- **会**：外键级联删除上表所列的关联**表记录**（与方式一相同）。  
- **不会**：删除 `data/user-data/.../project/{projectId}/` 目录，磁盘上会留下**孤儿文件夹**（章节 txt 等仍在）。  
- **不会**：删除知识库目录；`links_project` 行会被级联删掉，但知识库文件仍在。

因此运维与答辩表述应为：**一致性由应用层 `ProjectService` / `ChapterService` / `KnowledgeBaseService` 等在删库前先删目录保证**；不能假设「数据库级联 = 文件也删了」。

## 2. 运行环境与依赖

| 依赖 | 说明 |
|------|------|
| JDK | **17**（与 Spring Boot 3.x 一致） |
| MySQL | 本地或远程实例；库名需与配置一致（默认 **`wensi`**） |
| Maven | 项目自带 `mvnw`，一般无需单独安装 Maven |
| 浏览器 | 用于访问前端页面 |
| （可选）Navicat 等 | 查看库表与中文数据时，比部分终端更不易乱码 |

**端口说明**：后端监听端口以 `src/main/resources/application.properties` 中的 `server.port` 为准（**当前仓库默认为 `8081`**）。前端各页面中的 `API_BASE_URL` 需与该端口一致。

## 3. 数据库与敏感配置

### 3.1 不要在仓库中写死数据库密码

运行前通过环境变量注入，例如 **PowerShell（仅当前窗口生效）**：

```powershell
$env:DB_PASSWORD="你的MySQL密码"
```

### 3.2 启动时传参（不修改环境变量）

在项目根目录：

```powershell
.\mvnw spring-boot:run "-Dspring-boot.run.arguments=--spring.datasource.password=你的MySQL密码"
```

### 3.3 连接串与库名

确认 `spring.datasource.url` 指向你的实例与库名（默认库名为 **`wensi`**）。完整 JDBC 配置见 `src/main/resources/application.properties`。

### 3.4 其他环境变量（可选）

- **`EIDOS_SPEECH_API_KEY`**：全文朗读代理（解决 `file://` 打开页面时的跨域等问题），见 `application.properties` 中 `wensi.eidos-speech.api-key` 说明。

## 4. 初始化数据与用户文件目录

### 4.1 首次使用推荐顺序

与《快速启动指南》一致：**若尚无可用数据库与干净用户目录，可先执行「一键重置」（下文第 5 节），再启动后端（第 6 节）**。若已有数据库且仅需启动，可直接启动后端。

### 4.2 用户文件存储根路径

默认配置：

```properties
wensi.storage.base-path=${user.dir}/data/user-data
```

即：**从项目根目录启动时**，用户工作区位于 `data/user-data`。生产部署若工作目录不同，请通过配置或环境变量覆盖该路径，并保证进程对目录有读写权限。

## 5.（可选）一键重置数据库与 `data` 目录

**警告**：会重建 `wensi` 库并删除/重建 `data/user-data`，仅用于开发或明确要清空的场景。

在项目根目录执行（推荐显式传入密码，避免交互问题）：

```powershell
.\scripts\reset-dev-env.cmd -MySqlUser root -MySqlPassword "你的MySQL密码"
```

脚本行为摘要：

- 执行 `wensi_init_database.sql`（`DROP DATABASE IF EXISTS wensi` 后重建等）  
- 删除并重建 `data/user-data`  
- 结束时打印 `users` 表数量便于校验  

### 5.1 只重置其中一项

```powershell
# 只重置 data 目录（不动数据库）
.\scripts\reset-dev-env.cmd -SkipDatabase

# 只重置数据库（不动 data 目录）
.\scripts\reset-dev-env.cmd -SkipData -MySqlUser root -MySqlPassword "你的MySQL密码"
```

### 5.2 常见问题

- **脚本无法执行 / 未数字签名**：请使用仓库提供的 `scripts\reset-dev-env.cmd`；其会在**当前进程**内使用 `-ExecutionPolicy Bypass`，不修改系统全局策略。  
- **找不到 `mysql` 命令**：安装带 Client 的 MySQL，或将 MySQL 的 `bin` 加入系统 `PATH`，确保可执行 `mysql --version`。

## 6. 启动后端

在项目根目录（已设置 `DB_PASSWORD` 或使用上文传参方式）：

```powershell
.\mvnw spring-boot:run "-Dspring-boot.run.arguments=--spring.datasource.password=你的MySQL密码"
```

控制台出现类似 **`Started BackendApplication`** 即表示后端进程已起来。再次强调：**实际端口以 `application.properties` 为准（当前为 8081）**。

---

## 7. 访问前端

当前前端为静态资源，可直接用浏览器打开本地文件：

```text
file:///你的本机项目绝对路径/frontend/index.html
```

将 `你的本机项目绝对路径` 换为本机仓库根目录的绝对路径（注意 Windows 下路径中的 `/` 或编码习惯）。

**说明**：前端内 API 地址默认指向 `http://localhost:8081`，请与后端端口保持一致。若部署到域名或反向代理后，需批量调整 `frontend/js/api/client.js` 及各页面中的 `API_BASE_URL`，或通过后续改造为相对路径/可配置基址。

## 8. 生产部署建议（概要）

以下为常见上线方式思路，具体以贵方运维规范为准：

1. **构建后端**：使用 `mvnw package` 生成可执行 JAR，在服务器上配置 JDK 17、环境变量与 `data` 目录权限后运行。  
2. **静态前端**：将 `frontend/` 置于 Nginx 等静态资源服务下；配置同源或 CORS，并将 API 基址改为生产域名与 HTTPS。  
3. **数据库**：独立 MySQL 实例，开启备份与字符集 `utf8mb4`。  
4. **密钥与配置**：`DB_PASSWORD`、`EIDOS_SPEECH_API_KEY`、各 AI 相关密钥均通过环境变量或密钥管理服务注入，不入库不入仓明文。
