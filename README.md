## 项目简介（文思 WenSi）

**产品形态**：Web 应用，浏览器访问，无需安装客户端，跨平台使用。

**核心定位**：下一代智能写作引擎——面向学术研究者、技术文档撰写者、小说作者、笔记爱好者的专业级写作环境。

**Slogan**：写文如写「码」，让灵感一键「运行」。

### 解决的核心痛点（摘要）

- **思维断裂**：写作中难以快速找回过往笔记、术语或设定。  
- **缺乏结构化**：线性文档难以像代码一样模块化组织、跨处引用与版本管理。  
- **辅助不足**：通用聊天式 AI 难以深度贴合长篇创作上下文。  
- **引用追溯困难**：设定或术语多处使用时，难以统一查看定义与全部出现位置。

### 核心理念：像写代码一样写作

将编程领域成熟的工作流（结构化、跨文件引用、版本控制、自动化辅助）引入文字创作；把「函数调用」泛化为「知识引用」——在章节中写入知识条目名称或别名，系统可悬浮展示详情、跳转原文、追溯引用位置，使写作从线性码字升级为网络化知识组装。

### 产品价值主张（摘要）

| 角色 | 价值 |
|------|------|
| 学术研究者 | 文献笔记可复用，引用关系可追溯，减少重复阅读。 |
| 技术文档撰写者 | 术语一处维护、全局一致，降低文档与实现脱节风险。 |
| 小说作者 | 人物、地点、物品等可精准引用，利于长篇设定一致。 |
| 笔记爱好者 | 双向链接与个人知识图谱（进阶能力见需求说明）。 |

### 核心创新点（摘要）

| 创新点 | 说明 |
|--------|------|
| 函数式引用 + 悬浮预览 | 类似 IDE 调用提示，悬停即可查看知识条目概要。 |
| 反向链接追溯 | 列出引用当前条目的章节，类似「查找引用」。 |
| 多 AI 智能体协同 | 摘要、记忆、补全、聊天等场景可配置不同模型。 |
| 项目级长记忆 | 按项目维护 AI 长记忆，并压缩以防上下文溢出。 |
| Tab 补全式辅助 | 由用户主动触发符（如 `::`）再请求 AI，减少打断心流。 |
| 服务器端文件树 | 项目、知识库、章节对应真实目录，便于 Git 等版本管理。 |

### 技术架构概览

- **后端**：Spring Boot **3.5.13** + JDK 17，分层架构（Controller-Service-Repository）；Spring Security（密码 BCrypt 加密）+ JPA + MySQL；表结构由 Flyway 与初始化脚本管理。  
- **前端**：原生 HTML/CSS/JS，无第三方 UI 框架；通过 **HTTP REST** 调用后端（AI 聊天等为同步返回，界面有等待提示）。  
- **鉴权与隔离**：登录后用户信息存于浏览器 `localStorage`，接口携带 `userId`；后端在 Service 层通过 `getOwnedProject` 等校验归属。API Key 仅存服务端，由统一 HTTP 客户端调用大模型。  
- **存储**：数据库存元数据与关系；章节/知识正文等在服务器文件系统（默认 `./data/user-data/`，UTF-8）。  
- **AI**：用户自配 API Key（DeepSeek、OpenAI 兼容、DashScope 等）；实际调用走自封装 **OpenAI 兼容 HTTP 客户端**（`AiHttpClient`）。`pom.xml` 含 Spring AI 依赖，但 Chat 自动配置已排除，避免本地无密钥时启动失败。

### 核心工作流（用户侧）

1. 注册/登录，服务端创建用户工作区。  
2. 创建写作项目或知识库，生成对应目录。  
3. 在项目中编辑章节，引用已链接知识库中的条目。  
4. 在知识库中维护知识条目（正文、别名等）。  
5. AI 贯穿全流程：摘要、行内补全、侧边栏聊天、记忆管理等。

### 关键业务规则（摘要）

- 用户数据（项目文件、知识库、文本）存于**服务器端**；每用户独立根目录。  
- 项目与知识库可**多对多**链接；章节与知识条目引用关系支撑预览与反向追溯。  
- AI 能力基于用户自配 API Key；可按场景启用不同模型。  
- 每项目可有独立 AI 长记忆，并自动压缩管理。

### 功能模块索引（需求文档摘要）

- **用户与系统**：注册登录、主题与多 AI 模型配置、用户目录初始化、管理端等。  
- **知识库**：知识库/条目 CRUD、文件树、编辑器等。  
- **写作项目**：项目/章节 CRUD、IDE 风格编辑器、大纲、摘要、聊天与备注区等。  
- **引用机制**：项目-知识库链接、章节内联引用、悬浮预览、反向链接；进阶含知识图谱等。  
- **AI 协同**：章节摘要、Tab 补全、侧边栏聊天（REST）、记忆管理、改进建议、RAG 问答、文生图、识图、杂志风 PPT 等（需用户在设置中配置 Provider 并启用对应 Agent）。  
- **进阶与其它**：多级大纲、导出、RAG、管理后台、活动统计等（以实际实现与需求文档为准）。

### 非功能需求（验收关注点）

性能（文件树与编辑器流畅）、可用性（IDE 式布局与清晰反馈）、安全性（密码加密、服务端保存 API Key、`userId` 归属校验与用户目录隔离）、可靠性（统一异常处理；**经应用接口删除时**保证库表与项目/知识库目录一致）、可维护性与可扩展性（分层包结构、Agent 元数据可配置）。

### 删除与级联（简要）

- **经应用删除项目/知识库/章节**：Service 会先删对应磁盘目录，再删数据库记录；删项目时库表会通过外键级联清理章节、`links_project`、`links_chapter`、记忆、聊天等，**不会**删除已连接的知识库及其文件。  
- **仅在数据库中直接删行**：外键级联只影响表数据，**不会**自动删 `data/user-data` 下的文件夹，可能留下孤儿文件。详见 [部署指南.md](./部署指南.md) 第 1.6 节。

### 技术栈约束（与仓库一致部分）

- 后端：Spring Boot 3.5.13、JDK 17、包名 `com.wensi.backend`、Spring Security（密码编码）、JPA、MySQL、Flyway。  
- 前端：纯原生 HTML/CSS/JS，REST API（默认后端端口 **8081**）。  
- 部署：用户文件目录可配置（`wensi.storage.base-path`），默认指向项目下 `data/user-data`。
