第一次使用时启用方法：先【5】再【4】。如果已有数据库，使用【4】即可开启后端

【1】## **数据库密码**
不要写死明文密码；请在运行前通过环境变量注入：

PowerShell（当前窗口临时生效）：
【2】方式 A：在启动前设置一次环境变量（只对当前终端窗口生效）
```bash
$env:DB_PASSWORD="你的MySQL密码"
```

【3】方式B:或者启动时直接传参（不改环境变量）(在命令行输)：
```bash
.\mvnw spring-boot:run "-Dspring-boot.run.arguments=--spring.datasource.password=你的MySQL密码"
```
确认 spring.datasource.url 对应你的库（默认是 wensi）

【4】## **启动后端(第一次开始可以先使用下面的一键重置数据库和data目录命令，会建好数据库，然后再启动后端)**
在项目根目录执行：
.\mvnw spring-boot:run "-Dspring-boot.run.arguments=--spring.datasource.password=你的MySQL密码"
看到类似 Started BackendApplication 就算启动成功（默认端口 8080）
打开前端页面，在浏览器里输入本机绝对路径。
即：file:///你的本机项目绝对路径/frontend/index.html

请使用navicat或别的查看数据库，里面有中文，直接用终端显示会乱码。

【5】## **（可选）一键重置数据库和 data 目录**
在项目根目录执行：
```powershell
# 推荐：直接传密码（避免交互输入出问题）
.\scripts\reset-dev-env.cmd -MySqlUser root -MySqlPassword "你的MySQL密码"
```
说明：
- 会执行 `wensi_init_database.sql`（重建 `wensi` 库）
- 会删除并重建 `data/user-data`
- 会先 `DROP DATABASE IF EXISTS wensi` 再重建，并在最后打印 `users` 表数量用于校验

【6】只重置某一项（按需）：
```powershell
# 只重置 data 目录（不动数据库）
.\scripts\reset-dev-env.cmd -SkipDatabase

# 只重置数据库（不动 data 目录）
.\scripts\reset-dev-env.cmd -SkipData -MySqlUser root -MySqlPassword "你的MySQL密码"
```

常见问题：
- 如果提示“未对文件进行数字签名/无法在当前系统上运行该脚本”：请用上面的 `.cmd` 方式运行；它会用 `-ExecutionPolicy Bypass` 在**当前进程**内临时放行，不改系统全局设置。
- 如果提示找不到 `mysql` 命令：请安装 MySQL（带 Client），或把 MySQL 的 `bin` 目录加入系统 `PATH`，确保命令行能直接运行 `mysql --version`。
